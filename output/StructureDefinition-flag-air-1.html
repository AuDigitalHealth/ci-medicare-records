SD.12 StructureDefinition-flag-air-1

Australian Immunisation Register Notice [Draft]

Usage scenarios

The following are the expected usage scenarios this profile is intended to support:

  • Services Australia Medicare sends Australian Immunisation Register records to the My Health Record system

Implementation guidance

For the overarching usage scenarios in this implementation guide it is expected that:

  • the patient is identified with an IHI in subject.identifier, not with a reference to a Patient resource
  • notice text is sent in the text portion of code

SD.12.1 Formal Views of Profile Content

The official URL for this profile is: 

http://ns.electronichealth.net.au/ci/fhir/StructureDefinition/flag-air-1

The purpose of this profile is to define a representation of the Australian Immunisation Register (AIR) notice such as notice an individual has an exemption record or has finished a National Immunisation Program schedule.

Must Support

In the context of this profile Must Support SHALL be interpreted as follows:

  • The system SHALL be able to store and retrieve the element

Except for the following elements:

  • subject

for which it SHALL be interpreted as:

  • The system SHALL be able to take the element into account when performing processing

This profile builds on Flag.

This profile was published on Mon Mar 01 12:44:59 AEST 2021 as a draft by Australian Digital Health Agency.
